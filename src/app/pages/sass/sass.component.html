SASS

Sass es un lenguaje de programación 100% compatible con CSS pero con mas funcionalidades, dota a CSS de superpoderes de programación como variables, funciones, selectores anidados, operadores, modularización, etc. Actua como un preprocesador de CSS donde se escribe el códgo y despues se compila en un archivo CSS definitivo que es el que dara estilo a nuestro HTML. En la página oficial podemos ver toda la documentación y posibilidades que ofrece Sass

El archivo SASS debe tener formato .scss aunque también se permite el .sass pero funciona diferente a la hora de compilar.

Podemos incluir comentarios de una linea en el archivo .scss con // COMENTARIO...

Compilación

Hay varios metodos para compilar y transformar nuestro documento .scss en un archivo .css donde se represente todo el código escrito en SASS. Es recomendable tener Node.js instalado para realizar la conversión:

Live Sass Compiler
Extensión de VS Code donde va compilando cada vez que guardamos un archivo .scss en uno con el mismo nombre pero .css, hay que tenerlo activado abajo a la derecha clicando sobre Watch Sass. Todo el código debe ser escrito en el .scss no podemos escribir en el archivo compilado.
Terminal
Primero debemos asegurarnos que tenemos Node.js instalado y Sass instalado, este último lo podemos instalar escribiendo npm install -g sass en la terminal, con esto se instalará en Windows. Una vez instalado debemos escribir node-sass -w CARPETA SCSS -o CARPETA CSS con esto se estará compilando continuamente cualquier archivo que no tenga guión bajo delante "_NOMBRE" esto se hace para poder guardar mas archivos en la carpeta de origen.

Variables

$VARIABLE: VALOR;
Las variables se deben declarar con el simbolo $ y son muy utiles para cuando debemos utilizar un mismo valor (color, numero, lista, boolean, maps) multiples veces.

Maps

Declaración: $NOMBRE_MAP : ( "KEY_1" : VALOR_1, "KEY_2" : VALOR_2, "KEY_3" : VALOR_3 );
Llamada: map-get($NOMBRE_MAP, KEY);
Similares a los arrays de JS, son listas que contienen distintos valores (key) que despues los podemos utilizar como valor de una propiedad. Muy util para guardar valores del mismo tipo (fuentes, colores, tamaños) y despues llamarlos con map-get, de esta manera si queremos cambiar un valor que tenemos en distintos selectores solo hace falta cambiar el valor de la declaración del map.

Selectores anidados

SELECTOR_1 {{ '{' }}
    SELECTOR_2 {{ '{' }}
     ... 
    }
}
A diferencia de CSS, podemos anidar varios selectores, de esta manera el SELECTOR_2 solo se aplicara a todos los elementos de este tipo que esten dentro de un elemeto del tipo SELECTOR_1. Podemos escribir tantos selectores interiores como queramos con un máximo de 3 niveles de anidado. También podemos referirnos al selector padre como "&" para simplificar el código, muy útil para pseudo-selectores como &:hover.

@import

@import 'NOMBRE_ARCHIVO' ;
Es muy frecuente tener varios archivos .scss que permiten separar y organizar el código mejor que en un solo archivo, pero solo se puede compilar un .scss a .css, por este motivo se crean otros archivos .scss con una barra baja delante del nombre "_NOMBRE.scss" y despues los importamos al archivo compilador con esta función, cuando llamamos al archivo no hace falta poner la barra baja ni la terminación .scss. La modularización es muy util para separar archivos con variables, funciones, maps o incluso el codigo por partes de la estructura.

@function

@function NOMBRE (ARGUMENTOS) {{ '{' }}
    CODE_BLOCK 
}
Muy similar al funcionamiento en Javascript, podemos crear una función para que calcule o ejecute ciertas instrucciones o incluso para que nos devuelva un valor con el @return.

@mixin

@mixin NOMBRE (ARGUMENTOS) {{ '{' }}
    STYLE BLOCK
}
Muy parecido a @function pero mientras la anterior se usa mas para el calculo y retorno de valores, este se utiliza mas para agrupar estilos o compilar varias propiedades que despues queremos aplicar a distintos selectores, entonces solo con llamar al @mixin creado con @include NOMBRE (ARGUMENTOS); ya estaremos aplicando todas las propiedades del mixin.

@extend

@extend SELECTOR;
Sirve para copiar el mismo estilo de un selector a otro, escribimos @extend y el nombre del selector origen dentro del selector destino para copiar estilos, podemos aplicar mas estilos en el destino despues de esto.

Operadores

- + * /
En Sass podemos realizar operaciones para obtener el valor final siempre que sea entre la misma unidad métrica.

Colores

Sass tiene multiples formulas para modificar los colores, muy útil para cuando tenemos una referencia de un color y queremos sacar otro color en relación a este. Con estas funciones se pueden cambiar varios colores de una pagina modificando solo la referencia original. Guia completa en Sass.
