Princípios

Como incorporar código CSS

  • Inline - Permite añadir estilos CSS dentro de una etiqueta HTML con el atributo style="PRPIEDAD: VALOR, PROPIEDAD2: VALOR2..."
  • Internal- Se añade la etiqueta <style>...</style> en la sección head del documento HTML con los estilos a aplicar dentro.
  • External - Se añade la etiqueta y atributos <link rel="stylesheet" href="NOMBRE_ARCHIVO.css"> donde se linca el documneto CSS que va a aplicar estilo a dicho HTML.

Selector CSS

El selector hace referencia a la parte especifica del texto HTML a la que quieres aplicar un estilo concreto. Se pueden usar diferentes terminos como selectores:

  • Etiqueta HTML - Se pone como selector una etiqueta utilizada en nuestro código HTML, se aplicara el estilo a todas las etiquetas con este nombre.
  • #id - Se escribe # seguido del nombre del atributo id="NOMBRE" en la etiqueta que hemos identificado en HTML, solo afecta a una sola etiqueta.
  • .class - Se escribe . seguido de el nombre del atributo class="NOMBRE" en las etiquetas que hemos identificado en HTML, puede utilizarse en multiples etiquetas.
  • * - Hace referencia a todos los elementos del texto HTML, útil para dar mismo formato a todo el contenido (tipografia, espaciado, margenes, etc.)
  • [atributo] - Se refiere a todas las etiquetas que tengas este atributo, incluso podemos escribirlo con un valor exacto [atributo=valor]
  • :estado - Selecciona elementos a estilizar segun el estado en que se encuentren, se suelen combinar con los selectores anteriores para acotar mejor la selección. Los estados mas habituales suponiendo que van seguidos del elemento X:
    • :active (aplica cuando el elemento X esta activo, útil para links cuando son clicados).
    • :checked (cada elemento checkeado de los inputs de X).
    • :disabled (cada input deshabilitado de X).
    • :enabled (cada input habilitado de X).
    • :empty (cada elemento sin hijos dentro de X).
    • :first-child (cada X primer elemento dentro de otra etiqueta).
    • :first-child (cada elemento X que es el primer hijo de cualquier padre).
    • :focus (input de X que tiene el focus).
    • :hover (aplica cuando el mouse esta encima del elemento X).
    • :invalid (input de X con un valor incorrecto).
    • :last-child (cada último elemento X de cualquier padre).
    • :link (cada link sin visitar de X).
    • :optional (cada input de X que no es required).
    • :required (cada input de X que es required).
    • :valid (cada input de X con un valor correcto).
    • :visited (cuando se ha visitado ese elemento, útil para links ya abiertos).

      • Pseudo selectores con variables

    • :nth-child(n) (cada elemento X en la posición N del cualquier padre, donde N puede ser un número o even/odd).
    • :nth-last-child(n) (cada elemento X en la posición N de cualquier padre empezando a contar por el final, donde N puede ser un número o even/odd).

  • ::elementos - Pseudo elementos que pueden ser creados o seleccionados a partir de otros:
    • ::after (inserta algo despues de cada elemento X).
    • ::before (inserta algo antes de cada elemento X).
    • ::first-letter (La primera letra de cada elemento X).
    • ::first-line (la primera linea de cada elemento X).
    • ::seleciton (la parte que el usuario selecciona).

Todos estos elementos (excepto *) son combinables entre ellos, tanto para seleccionar mas de un elemento y aplicar mismo estilo separandolos por comas ",", por ejemplo: h1, h2, h3 (aplica lo mismo para todos) o para acotar la selección con mas de un selector sin comas, por ejemplo: p .caption (todos los paragrafos con class="caption"), .class1 .class2 (elementos con 2 clases), div p (paragrafos dentro de divs), div>p (paragrafos con parentesco directo de div, solo padre no abuelo), div + p (primer p dentro de div) a:hover (links con mouse encima), etc.
Si aplicamos estilo a un elemento sin especificar que es solo ese elemente y este tiene dentro otro elemento, este último recibirá el mismo estilo hererado hasta que le indiquemos lo contrario.
Todos los navegadores aplican un estilo por defecto a HTML incluyendo tipografia, margin, padding, tamañanos, etc. Puede ser conveniente utilizar * para desactivarlo o aplicar el estilo que mas nos convenga a toda la página.

Puede ser que los navegadores necesiten un prefijo para saber interpretar el código CSS, sobretodo las últimas propiedades incorporadas como CSS3. Hoy en dia la mayoria pueden trabajar sin estos prefijos pero nos podemos encontrar el caso en que se necesiten. También se debe incorporar cada propiedad con su prefijo para que el mismo código sea legible en todos los navegadores.

Unidades de medida

Para definir el valor de una propiedad CSS permite la escritura de varias unidades de medida para distintos valores:

Holi. Lorem ipsum dolor sit amet, consectetur adipisicing elit. Suscipit alias laudantium in? Rem, eaque et! Pariatur vitae ratione unde adipisci? Nostrum, reprehenderit molestiae non rerum et temporibus molestias quia itaque.

Unidades métricas
Permite definir una distancia entre dos puntos para personalizar el largo, ancho, borde, espacio o cualquier otra propìedad con valor númerico. Distinguimos las siguientes unidades mas habituales:

Unidades absolutas, números independientes que expresan un valor exacto:
  • cm - Centimetros
  • mm - Milímetros
  • in - Pulgadas (1in = 96px = 2.54cm)
  • px - Pixeles (puede variar la perspectiva segun la resolución o dispositivo donde se visualice)
  • pt - Puntos (medida habitual de texto.)
  • ch - Characteres, medida en caracteres, muy útil para medir largos de linea (width) por ejemplo.
Unidades relativas, van en función de otras unidades, por lo que pueden variar el valor segun la referencia:
  • em - Depende de la medida del font-size del elemento padre, 1 em = font-size x1
  • rem - Depende de la medida del font-size básica de la pagina (relativo al tamaño de fuente raíz del html.), 1 rem = font-size x1
  • % - Depende de la medida donde se encuentre la caja, si no tiene parentesco ira en función de la medida máxima de la página/navegador
  • vh - ViewPort Height, hace referencia a la altura de la parte visible del navegador, acepta del 0 al 100 como si se tratará de %.
  • vw - ViewPort Widht, ancho visible del navegador donde 1vw equivale a 1%.
  • calc() - Permite hacer calculos con los valores dentro de los parentesis para determinar el valor resultante.
  • min(...) - Permite establecer un minimo para medidas cambiantes como anchos, largos, etc. Se aceptan varios valores y se elegira siempre el menor. Ej. min( 20%, 100px, 4rem )
  • max(...) - Permite establecer un máximo para medidas cambiantes. Acepta mas de un valor eligiendo el que sea siempre mas grande. Ej. max( 80%, 500px, 50vw ).
  • auto - Igual que % depende de su parentesco, en este caso reparte el mismo valor si hay mas de una propiedad o equivale al valor más céntrico (útil cuando asignamos un mismo valor a multiples propiedades/lados).
    • *min() y max() permiten mas de un valor entre parentesis. Ejemplo width: min(500px 70%); escogerá siempre la mínima medida en función del ancho navegador, si 500px es menor que 70% entonces escogerá 500px como medida mínima. Lo mismo para max() pero escogiendo la medida mas grande.
Colores
Siguiendo varios estandares, hay distintas maneras de expresar el valor del color que queremos asignar:
  • Hexadecimal - #RRGGBB asigna dos unidades de valor del 0 a la F para cada color básico.
  • Hexadecimal con transparencia - #RRGGBBAA asigna dos unidades de valor del 0 a la F para cada color básico y los dos últimos para el canal alpha.
  • RGB - rgb (R,G,B) asigna un valor del 0 al 255 para cada color básico, también se puede escribir con %.
  • RGBA - rgba (R,G,B,A) asigna un valor del 0 al 255 para cada color básico, y por último la transparencia con un valor del 0.0 (100% transparente) al 1.0 (100% opaco).
  • HSL - hsl(HUE, SATURATION, LIGHTNESS) teniendo en cuenta la rueda de color, asigna un grado de 0 al 359 para la tonalidad, y un porcentage para la saturación y brillo(0% es negro y 100% es blanco).
  • HSLA - hsla(HUE, SATURATION, LIGHTNESS, ALPHA) igual que HSL pero añadiendo transparencia con valores del 0.0 al 1.0.
  • Predefinidos - colores ya predefinido por CSS que se definen con su nombre, hay 140 disponibles (red, green, lightgrey, darkblue, etc.).
  • linear-gradient(DIRECCIÓN, COLOR1 STOP, COLOR2 STOP, COLOR3 STOP...) - permite crear degradados lineares, por defecto la dirección va de top a bottom a no ser que le indiquemos lo contrario con bottom/right/left o Xdeg. El valor de STOP se indica en números px o % y definde donde acaba dicho color, dos colores con el mismo valor crearan un cambio abrupto sin degradado. Hay una variante que añade la palabra repeating- al inicio y hace que se repita el degradado hasta completar el elemento a colorear.
  • radial-gradient(X Y, SHAPE, COLOR1 STOP, COLOR2 STIOP, COLOR3 STOP,...) - permite crear degradados radiales, debe indicarse la posicion X Y del centro con px o %, la forma puede ser circle o ellipse y los colores con la métrica de finalización. Hay una variante que añade la palabra repeating- al inicio y hace que se repita el degradado hasta completar el elemento a colorear.

Estilos para texto

font-family: Arial, Roboto, Serif, Sans Serif;
Aplica tipografia al selector, suele indicarse varias fuentes por si el navegador no puede aplicar alguna, también se suelen añadirse fuentes familiares al final que incluen un grupo de tipografias parecidas para dar mas opciones al navegador. En caso de no poder aplicar ninguna, cada navegador tiene su propia tipografia por defecto. También se pueden incluir las fuentes con la url y etiqueta <link...> en el meta o la regla @font-face importando el archivo.
font-size: 12px
Tamaño del texto, puede ser definido en medidas (small, medium, large), en pixeles, cm, %, in, rem (en función del estandar), em, etc.
font-weight: lighter
Anchura de la letra, puede definirse en medidas (lighter, normal, bold) o en centenas del 0 al 1000 donde 400=normal.
font-style: italic
Permite modificar el texto con letras en formato italic, normal o oblicuas
font-variant: small-caps
Permite modificar el texto a todo mayusculas pequeñas (small-caps) o normal que esta por defecto.
font: STYLE VARIANT WEIGHT SIZE FAMILY
Abreviatura de varios estilos "font" en una unica linea, también se pueden aplicar estilos predefinidos como: caption, icon, menu, message-box, status-bar, etc.
color: lightblue
Pinta el texto del color indicado con nombres (red, blue, grey), valor hexadecimal (#000FFF) o valor rgb(255,100,100).
text-align: center
Centra el texto a la parte que le indiquemos, admite valores como: left, right, center, start, end y justify.
line-height: 100px
Alinea el texto en medio de la medida indicada segun el height del contenido de la caja inline (no tiene en cuenta padding ni margin).
text-decoration: underline
Permite añadir una derocación al texto, acepta valores como: blink, overline, underline, line-through
text-indent: 10px
Permite añadir sangria al comienzo de la primera linea de texto, acepta valores numericos en px, cm, pt, em, %.
text-shadow: X Y BLUR COLOR
Dibuja una sombra con los 4 valores indicados, 3 primeros numericos para eje de X, Y y desenfoque y el último valor para el color de la sombra SHADOW
text-transform: uppercase;
Permite transformar texto en mayusculas con varios valores:
  • capitalize (primera letra de cada palabra mayuscula)
  • uppercase (todas las letras mayusculas).
  • lowercase (todas las letras minusculas).
text-overflow: ellipsis
Indica el comportamiento del texto que sobresale. Puede ser clip, ellispsis o string.
column-count: 2;
Separa el texto en el numero de columnas indicadas.
column-gap: 1.5rem;
Especifica el espacio entre columnas.

*Hay mas propiedades para columnas disponibles. Ej. column-width, column-rules, column-fill, column-span, etc.

letter-spacing: 2px;
Permite modificar el espaciado entre carácteres incluso con valores negativos, afecta a la legibilidad, acepta numeros en px, pt, cm, mm, etc. o normal
word-sapcing: 5px;
Modifica espacio entre palabras con valores númericos en px, pt, cm, mm, etc.
white-space: pre-wrap
Permite interpretar los espacios en blanco y los saltos de linea de otra manera:
  • nowrap (ignora multiples espacios en blanco y realiza saltos de linea en break).
  • pre (conserva todos los espacios en blanco y preserva los del navegador).
  • pre-line (ignora multiples espacios en blanco).
  • pre-wrap (preserva espacios del navegador y muestra todos los espacios en blanco).
word-wrap: break-word;
Permite a las palabras largas que no caben por el margen derecho separarse para continuar en la siguiente linea. Acepta valores: normal (por defecto) o break-word (separación habilitada).
@font-face {{ '{' }} font-family: NOMBRE; src: url(NOMBRE_ARCHIVO.ttf);{{ '}' }}
Permite importar fuentes tipográficas externas. Debemos tener el archivo .ttf/.otf/.woff/.woff2 en la carpeta, le asignamos un nombre de font-family: ... ; que despues podemos utilizar al asignar una tipográfica con esta misma propiedad. Por último indicamos la ruta y/o nombre del archivo con src: url(...);

Propiedades de cajas

Cada elemento de HTML esta representado como una caja en la pàgina, gracias a CSS podemos modificar todas las partes y posicionamiento de estas cajas. Distinguimos 4 partes diferentes de la caja: margin, border, padding y content. Cada una de ellas se puede alternar por los cuatro lados o por cada lado por separado refiriendose a top, right, bottom y left.

width: 100px
Ancho de la caja, afectando unicamente al content, acepta valores en px, %, cm, in, etc. Se puede añadir el prefijo max- para indicar la anchura máxima permitda y el prefijo min- para la medida mínima.
height: 50%
Alto de la caja, afectando unicamente al content, acepta valores en px, %, cm, in, etc. Se puede añadir el prefijo max- para indicar la altura máxima permitda y el prefijo min- para la medida mínima.
padding: 1cm
Espacio que separa el content del border, acepta valores en px, %, cm, in, etc. Podemos poner un unico valor que sera igual para los 4 lados, dos valores que definiran primero top y bottom y despues left y right, o cuatro valores individuales para top, right, bottom y left respectivamente.
border: WIDTH STYLE COLOR
perimetro de la caja el cual se puede hacer visible y colorear, por defecto no hay, para que se vea hay que indicar la anchura(px, %, cm, in, etc.), el estilo (solid, dashed, dotted, double, etc.) y el color (rgb, hsl, hex o nombre).
margin:
Espacio de margen que hay despues del border y que separa del resto de cajas, acepta valores en px, %, cm, in, etc. Podemos poner un unico valor que sera igual para los 4 lados, dos valores que definiran primero top y bottom y despues left y right, o cuatro valores individuales para top, right, bottom y left respectivamente.

*Estas 3 ultimas propiedades podemos modificarlas aplicando solo sobre un lado, tan solo con añadir "-" y el lado que nos interese (top, rigth, bottom, left). Incluso border podemos podemos actuar con una de las 3 propiedades y un lado, ejemplo: border-width-bottom.

border-radiusTOP-LEFT TOP-RIGHT BOTTOM-RIGHT BOTTOM-LEFT;
Permite redondear las esquinas del borde dibujado con valores en px o %. Se puede indicar un único valor para todas las esquinas o 4 distintos con el orden del ejemplo. Permite crear circulos si width mide igual que height e indicamos la mitad de border-radius.
background-color: lightblue;
Modifica el color del fondo de la caja, afectando al content y padding.
background-image: url(NOMBRE_ARCHIVO.jpg);
Establece una imagen como fondo de la caja, puede aceptar mas de una imagen y se mostraran en orden. También acepta valores como: linear-gradient(), radial-gradient(), repeating-linear-gradient(), repeating-radial-gradient().
background-repeat: repeat (X,Y);
En caso de tener una imagen de fondo podemos repetirla tantas veces como queramos en cada eje, también acepta valores como: repeat-x(X), repeat-y(Y).
background-position: left center;
Indica donde debe empezar la imagen de fondo, se puede poner con nombre de los lados o X% Y%, Xpos Ypos.
background-size: cover;
Indica las medidas que debe tener el fondo establecido, cover hace que cubra todo el content y padding. También acepta valores númericos indicados para X e Y de cada eje (%, px, pt, cm, etc.) o contain que hace que la imagen sea completamente visible independientemente de las medidas de la caja.
background-clip: content-box
Define hasta donde debe ocupar el color o imagen del background de la caja. Acepta los siguientes valores: border-box (incluyendo border), padding-box (incluyendo padding) o content-box (solo content).
background: COLOR IMAGE POSITION / SIZE REPEAT CLIP ATTACHMENT;
Permite establecer varios valores en una misma propiedad, no hace falta definirlos todos. Si se indica la propiedad size de debe añadir el slash "/" para separarlo de position.
box-sizing: border-box;
Indica como se calculan el width y height de las cajas del elemento seleccionado. Muy útil para definir mismo metodo para todo. Acepta valores: content-box (por defecto, width y height solo definen el content) o border-box (witdh y height incluyen content, padding y border).
box-shadow: X Y BLUR DESPLAZAMIENTO COLOR
Permite añadir sombra a la caja del elemento seleccionado, X e Y siempre son valores obligatorios. Por defecto el resto de valores son 0 y negro. También permite crear sombras interiores insertando el valor inset antes del resto de valores. Podemos crear multiples sombras separandolas por comas.
outline: WIDTH STYLE COLOR
Contorno que se dibuja despues del borde del elemento, puede sobreponerse a elementos contiguos.
opacity: 0.5;
Aplica transparencia a toda la caja seleccionada, permite valores del 0.0 a 1.0 o también %, donde va de transparente a opaco.

Propiedades de las tablas

Indica si los bordes que separan las celdas y el borde exterior de la tabla se juntan o no, puede ser: separate (por defecto) o collapse (junto).
border-spacing: 10px;

caption-side bottom;
Define donde debe aparecer la descriptión (etiqueta caption) de la tabla, puede ser bottom o top.
empty-cells: hide;
Indica si los bordes de las celdas vacias deben mostrarse o no, puede ser show (por defecto) o hide.

Indica el metodo para medir la anchura de las columnas y la tabla, en valor auto (por defecto) se mide en función al contenido, en modo fixed se mide en función de la anchura de la tabla o columna o por el ancho de la primera fila.
list-style: TYPE POSITION IMAGE;
Permite añadir las 3 propiedades anteriores en una sola .

Posicionamiento y disposición

Cada elemento de HTML esta representado como si fuera una caja con contenido, algunas tiene comportamiento por defecto block y ocupan todo el ancho de la página obligando al siguiente elemento posicionarse debajo, otras tiene por defecto inline ocupando solo el volumen del contenido y deja espacio a su derecha para el siguiente elemento. Con esta propiedad podemos modificar el comportamiento de las cajas.

display: inline;
Propiedad que modifica el comportamiento de la caja respecto al resto, admite distintos valores:
  • none - oculta el elemento pero
  • block - empieza en una nueva linea y ocupa todo el ancho de la pagina impidiendo otro elemento a su derecha, permite modificar propiedades de la caja (widht, height, margin...)
  • inline - ocupa el minimo volumen de su contenido y permite posicionar los siguientes elementos a su derecha, si un elemento inline no termina por el margen derecho continuara en la siguiente linea. No permite modificar propiedades de la caja, ya que se calcula por su contenido únicamente.
  • inline-block - se comporta igual que inline per si permite modificar propiedades de la caja.
  • flex -
  • grid -
position: fixed;
Define el posicionamiento de la caja en la página de distintas maneras:
  • static (por defecto) - posición en orden segun su diplay.
  • absolute - se posiciona respeto al previo elemento no-static. Permite valores top, right, bottom, left.
  • fixed - posición fija en el navegador sin verse afectado por el scroll. Permite valores top, right, bottom, left.
  • relative - posición static pero permite valores top, right, bottom, left. Con estas propiedades se modificará su posición respeto a la original (puede superponerse a otros elementos).
  • sticky - posición static hasta baja el scroll y se comporta como fixed. Conserva su posición hasta que el scroll choca con las medidas top, right, bottom o left indicadas y queda en posición fixed dentro del espacio de su contenedor. Si se aplica a un elemento repetido en la página, solo se arrastrará el último elemento

*Las propiedades top, right, bottom y left deben escribirse despues de position:...; y modifican la posicion original indicada dejando el espacio definido en el lado de la propiedad.

float: left;
Posiciona los elementos hacia un lado permitiendo que los siguientes elementos ocupen el espacio del otro lado libre (parecido a display: inline;). Se pueden encadenar varios elementos con esta propiedad y ocuparan todo el ancho de la misma linea hasta no caber mas. Permite valores: left, right y none (por defecto). Muy útil para imagenes donde queremos mostrar texto a su lado.
clear: both;
En combinación con la propiedad anterior float:... indica donde no debe mostarse mas elementos, dejando ese lado despejado. Permite valores: right, left o both.
visibility: hidden;
Permite esconder el contenido, pero seguirá ocupando el mismo espacio por lo que se verá un espacio en blanco, puede ser visible (por defecto) o hidden.
overflow: scroll;
Indica el comportamiento cuando una caja es mas grande que su contenedor (suele ocurrir si no se indica width i height). Acepta los siguientes valores:
  • visible - el contenedor crece hasta que cabe todo el contenido del elemento interior.
  • hidden - se muestra únicamente el contenido visible, el resto queda escondido.
  • scroll - se habilita una barra de scroll para poder visualizar todo el contenido.
  • auto - en caso de que el contenido no quepa, se habilita una scroll-bar.
z-index: -1;
Determina el comportamiento con elementos superpuestos, solo afecta a elementos con position: absolute/relative/fixed/sticky. y elementos flex. Admite valores númericos (sin unidad de medida), el numero mas alto ira por delante del mas pequeño, por defecto tiene valor 0 todos los elementos y los ultimos elementos escritos superponen a los anteriores si no se indica lo contrario.

Flexbox

Guia completa de Flexbox en CSS-TRICKS

Modo de distribución de elementos donde se muestran segun un eje principal y uno secundario. No existe top, center, right or left. Se debe evitar medidas de margen, ya que Flex se encarga de ello. En la mayoria de situaciones tendremos un contenedor donde se distrubuyen los distintos elementos a mostrar, utilizaremos unas propiedades distintas para los elementos padres (contenedores) y para los elementos hijos (contenidos). Al contenedor padre le indicaremos la propiedad display: flex; para definir este tipo de distribución.

Propiedades para contenedores:

flex-direction: column;
Determina como se van a distribuir los hijos y como se van a ir mostrando. Valores aceptados: column (acumula elementos en columna de arriba a abajo), row (por defecto, acumula elementos en la misma fila de izquierda a derecha), column-reverse (de abajo a arriba) y row-reverse (de derecha a izquierda).
flex-wrap: wrap;
Define como se deben comportar los elementos cuando no caben en la misma linea, con no-wrao (por defecto) los elementos cambian su tamaño para encajar en la misma linea, con wrap los elementos conservan siempre su tamaña y hace salto de linea cuando se requiere. Por último wrap-reverse conserva tamaña y hace salto de linea en sentido contrario.
flex-flow: FLEX-DIRECTION FLEX-WRAP;
Permite indicar las dos propiedades anteriores en una misma. (Ej. flex-flow: cloumn wrap;)
justify-content: space-around;
Define el metodo en que se van a alinear los elementos contenidos. Acepta los siguientes valores:
align-items: stretch;
Alinea los elementos hijos en el eje contrario dentro del espacio disponible en el contenedor padre. Acepta los siguientes valores:
align-content: space-around;
Define como se distribuye el conjunto de elementos dentro del contenedor, muy útil cuanto tenemos saltos de linea. Acepta los siguiente valores:
gap: 10px;
Define el espacio entre los elementos en ambos ejes, no tiene en cuenta espacio entre elemento y borde del contenedor. Indica el espacio minimo que tiene que haber, si otra propiedad como justify-content: space between; deja mas espacio no se tiene en cuenta el gap. También existe la variedad para indicar gap en un solo eje: column-gap i row-gap.

Propiedades para los contenidos:

order: 5;
Define que orden van a tener los elementos en el eje principal y secundario. Se puede indicar individualmente un valor númerico a cada elemento y estos se ordenaran de menor a mayor. Acepta valores negativos.
flex-grow: 4;
Reparte el espacio disponible entre los hijos que tengan esta propiedad. El valor númerico indica las proporciones a ocupar, si dos elementos tienen el mismo valor se repartiran el espacio sobrante a partes iguales, si uno tiene un valor mayor, se suman todas las cantidades y se asignan las proporciones de los elementos segun su valor. No acpeta números negativos.
flex-shrink: 2;
Al contrario que flex-grow, define la capacidad para encogerse.
flex-basis: 2rem;
Define el tamaño original de un elemento en el eje principal, es preferible determinar el tamaño de un elemento hijo con esta propiedad que con width o height. Acepta valores númericos, content (tamaño se ajusta al contenido) o auto (se ajusta por propiedad width o height o al contenido si no tiene width o height).
flex: FLEX-GROW FLEX-SHRINK FLEX-BASIS;
Define las 3 propiedades anteriores en una misma propiedad. Se puede resumir en valores como: none (0 0 auto, No es flexible), auto (1 1 auto, si es flexible) o number (number 0 0, ideal para aplicar proporción y mantener flexibilidad)
align-self: center;
Permite alinear en el eje secundario un solo elemento diferente al resto. Acepta valores como: auto | flex-start | flex-end | center | baseline | stretch

Grid

Guia completa en: CSS-tricks y comparativa con Flexbox aquí

Distribución de los elementos en forma de cuadricula o rejilla, a diferencia de flexbox, tendremos dos ejes principales donde posicionar los elementos y 2 dimensiones. No sustituye a flex, sino que lo complementa. Al igual que flexbox se diferencia entre el container (padre) y los elementos contenidos (items hijos), también diferenciamos las lineas que separan los elementos y los bordes del container (Grid line) que pueden ser horizontal o verticales. Grid Track es una columna o fila donde tenemos todos los Grid cell (item hijo). Por otro lado, también podemos definir un Grid area que es un conjunto de grid cells en filas y columnas contiguas.
Para empezar a crear un grid debemos indicar display: grid; al elemento container y configurar las siguientes propiedades diferentes para container e items:

Propiedades para container:

grid-template-columns: repeat (5, 1fr)
Define el numero de colummnas y el ancho de cada una. Se puede indicar de varias maneras:
  • Xpx Xpx Xpx... - crea tantas columnas como valores y le da el ancho indicado respectivamente, acepta valores como px, fr (fracción), %, auto (se adapta al contenido).
  • repeat(X, 1fr) - crea X columnas con el ancho indicado, puede ser 1fr o Xpx. Acepta mas de un valor y lo repite constantemente. X puede ser un numero o "auto-fill" (llena tantas columnas como quepan en su tamaño minimo y puede dejar celdas vacias) o "auto-fit" (llena el espacio con tantos items haya sin dejar vacios).
  • minmax(10px, 1fr) 3fr - crea las columnas indicadas en el último valor pero con un min. y un max. limitados entre parentesis. También acepta valores "min-content" o "max-content" que limitan el tamaño al contenido del item.
  • [column1] 1fr - asigna un nombre a la linea [NOMBRE] y un valor. El nombre se puede usar posteriormente para. En la última columna no hace falta poner valor de anchura.

*Se pueden combinar las diferentes formas de indicar los valores. Ej. repeat(auto-fit, [col-big] minmax(100px, 1fr) [col-md] minmax(50px, 1fr) [col-small] minmax(min-content, 1fr));

grid-template-rows: 50px 1fr auto 10%;
Define el numero de filas que va a tener nuestro grid, acepta los mismos valores que la propiedad anterior.
grid-template-areas:
"header header header"
"item1 item1 aside"
"item2 item3 aside"
"footer footer footer"
Crea el grid posicionando cada Grid-area en el sitio indicado. Es necesario definir el nombre de cada elemento con grid-area . Este metodo crea de forma sencilla una cuadricula al gusto. Podemos utilizar "." para definir una celda vacia.

*Si por alguna razón tenemos mas elementos que celdas configuradas, por defecto se ajustaran al tamaño de su contenido siguiendo la cuadricula. También podemos configurar las propiedades grid-auto-columns y grid-auto-rows que definiran como deben de ser las celdas extra en caso de tener mas items de los que quepan. Acepta los mismos valores que grid-template-columns. También podemos definir el comportamiento de los elementos extra con grid-auto-flow donde admite valores row (por defecto, los elemntos extra se añadiran creando mas filas) o column (los items extra se añadiran creando nuevas columnas). A parte de row/column podemos añadir dense que impide que queden celdas vacias rellenando con los siguientes elementos, por defecto si no se indica puede que queden celdas vacias segun la configuración del grid.
Se diferencia entre configuración grip "Explicit" que se refiere a los elementos tenidos en cuenta en al configuración y a "Implicit" que son los items extra que no hemos tenido en cuenta.

grid-template: GRID-TEMPLATE-AREAS GRID-TEMPLATE-ROWS GRID-TEMPLATE-COLUMNS
propiedad donde se pueden indicar las 3 anteriores en la misma.
column-gap: 10px;
Indica la sepración entre los elementos por columnas, no tiene en cuenta la distancia al borde del container.
row-gap: 15px;
Defina la separación entre filas sin tener en cuenta la distancia al borde del container.
gap: 10px;
Define el espacio entre todos los elementos, con un solo valor damos la misma distancia para row-gap y column-gap a la vez. Si ponemos dos valores seguidos, el primero indica row-gap y el segundo column-gap.
justify-itmes: center;
Alinea los items en el eje horizontal. Acepta los siguientes valores:
  • start - Alinea todos los items al lado izquierdo de celda.
  • end - Alinea los items al lado derecho de su celda.
  • center - Centra los items dentro de su celda.
  • stretch - Rellena todo espacio disponible de todas las celdas de izquierda a derecha.
align-items: stretch;
Justifica los items en el eje vertical dentro de su espacio disponible en cada celda. Acepta los siguientes valores:
  • start - Alinea los items al lado superior de su celda.
  • end - Alinea los items al lado inferior de su celda.
  • center - Alinea los items al centro del eje vertical dentro de su celda.
  • stretch - Rellena todo el espacio vertical de cada celda.

Propiedades para items:

grid-column-start: column1;
Asigna la linea de columna donde debe empezar ese elemento. Acepta los siguientes valores:
  • NOMBRE - el nombre asignado en la propiedad grid-template-column del elemento contenedor.
  • X - numero de la linea, empezando des de 0 en el borde izquierdo.
  • span X - se expandirá tantas celdas como marque el valor X.
  • span NOMBRE - se expandirá hasta llegar a la linia con dicho NOMBRE.
  • auto - se coloca automaticamente segun lo definido por otros elementos o el contenedor.
grid-row-start: span 2;
Indica en que linea de fila debe empezar dicho elemento. Acepta los valores anteriores.
grid-column-end: 3;
Define el numero de linea de columna donde finaliza este elemento. Acepta los mimos valores anteriores.
grid-row-end: auto;
Indica en que linea de fila debe terminar dicho elemento. Acepta los valores anteriores.
grid-column: GRID-COLUMN-START / GRID-COLUMN-END;
Define la linea de la columna donde empieza y donde termina en una misma propiedad, separadonlo por "/".
grid-row: GRID-ROW-START / GRID-ROW-END;
Indica en que linea de fila debe empezar y en cual debe terminar.
grid-area: NOMBRE ROW-START COLUMN-START ROW-END COLUMN-END
Define el nombre para este item, dicho nombre es util para la propiedad grid-template-area del container. También se puede indicar el inicio y fin de columna y fila para este item.
order: 1;
Igual que flex, se pueden reordenar los items con esta propiedad, se ordenaran de menor a mayor. Acepta numeros negativos.
justify-self: start;
Alinea este único item a diferencia del resto en su eje horizontal. Admite valores como start, end, center o stretch.
align-self: center;
Alinea este único item en su eje vertical. Puede ser: start, end, center o stretch.
place-self: JUSTIFY-SELF ALIGN-SELF;

En general, Grid es utilizado para crear las estructuras y layout y Flexbox se utiliza para alinear los elementos interiores de los items grid que lo requieran.

Animaciones

Las animaciones permiten cambiar las propiedades de los elementos en un intervalo de tiempo e incluyendo suavidad de movimiento.

Transitions

Muy útiles para cuando definimos diferentes valores a un mismo elemento en varios estados y para evitar que cambie bruscamente y añadir un movimiento mas suave.
transition-property: border
Propiedad que queremos animar, si no se indica ninguna se aplicara a todas las propiedades modificadas.
transition-duration: 2s
Duración de la animación, puede ser en segundos (s) o en milisegundos (ms).
transition-timing-function: ease-in-out
Define la curva de velocidad. Permite varios valores como: ease (por defecto), linear, ease-in, ease-out, ease-in-out, step-start, step-end. O incluso con curvas bezier personalizadas con cubic-bezier(n,n,n,n) donde las N puede tener un valor entre 0 y 1.
transition-delay: 3s
Permite añadir un tiempo de espera para iniciar la transición. Valores en segundos (s) o milisegundos (ms).
transition: PROPERTY DURATION TIMING-FUNCTION DELAY, ...
Abreviatura para simplificar todas las propiedades anteriores en una sola, permite añadir mas de una transición separando por comas.

Animations

Las animaciones permiten crear movimiento y cambiar las propiedades de un elemento cuando queramos.
Para crear animaciones hay que declarar primero los @keyframes que despues podremos incorporar a todos los elementos que queramos.

@keyframes NOMBRE
Declaración de los pasos que va a tener la animación junto a las propiedades y valor a animar. Se pueden crear tantos pasos como se deseen. Los pasos se pueden declarar con from y to o de 0% a 100% y cada paso va seguido de propiedades y valor a modificar en ese instante enter corchetes {{ '{' }}...}. Ej.

@keyframes mymove {{ '{' }}
  0% {{ '{' }}top: 0px; background: red; width: 100px;}
  50% {{ '{' }}top: 50px; background: orange; width: 100px;}
  100% {{ '{' }}top: 200px; background: yellow; width: 300px;}
}

Una vez hecha la declaración de los keyframes podemos utilizar esta animación para aplicarla a cualquier selector con las siguientes propiedades:

animation-name: NOMBRE
Se indica el nombre que hace referencia al @keyframes que queremos utilizar.
animation-duration: 2s;
Define la duración entre el primer y el último keyframe. Acepta valores en segundos (s) o milisegundos (ms).
animation-delay: .5s
Define el tiempo previo a iniciar la animación. Acepta valores en segundos (s) o milisegundos (ms).
animation-direction: reverse;
Define el orden a seguir de los keyframes. Puede ser: normal (por defecto), reverse, alternate, alternate-reverse.
animation-fill-mode: forwards;
Define que estilo debe aplicarse al elemento antes y/o despues de la animación. Puede ser forwards (se queda con el último keyframe), backwards (esta como el primer keyframe hasta empezar) o both (esta como el primero y se queda como el último).
animation-iteration-count: infinite;
Define el número de repeticiones que debe reproducirse la animación, puede ser un número o infinito.
animation-play-state: paused | running;
Útil para pausar o reaunudar las animaciones con cambios de estado del selector o eventos (:hover, :active, scroll, etc.).
animation-timing-function: ease-out
Define la curva de velocidad. Permite varios valores como: ease (por defecto), linear, ease-in, ease-out, ease-in-out, step-start, step-end. O incluso con curvas bezier personalizadas con cubic-bezier(n,n,n,n) donde las N puede tener un valor entre 0 y 1.
animation: NAME DURATION TIMING-FUNCTION DELAY ITERATION-COUNT DIRECTION FILL-MODE PLAY-STATE
Abreviatura para definir todas las propiedades en una sola.

Otras propiedades

@media MEDIATYPE and (MEDIAFEATURE) {{ '{...}' }}
Define reglas para modificar los esticlos CSS segun los condicionales entre parentesis, siempre que sean true se aplicar los estilos contenidos donde solo sustituiran a los originales en caso de repetirlos. También se puede definir el Mediatype que indica a que dispositivos se aplica la regla, por defecto se aplica a todos, podemos limitar a print (impresoras), screen (pc, portatil, tablet, movil, etc) o speech (screenreaders). Entre parentesis debemos indicar el condicional o mediafeature, valores mas comunes:
  • min-width: Xpx - cuando la anchura minima sea X, también valido para min-height.
  • max-height: Xpx - cuando la altura máxima sea X, también válido para max-width.
  • orientation: landscape - cuando el viewport es mas ancho que alto, "portait" seria el caso contrario.
  • resolution: X - resolución del dispositivo reproductor en dpi o dpcm. Se puede añadir el prefico min- o max-
Podemos añadir auxiliares delante del Mediatype para cambiar el significado como not (invierte el significado) o only (usado para navegadores antiguos), en los Mediafeatures, ademas de not podemos utilizar también and (para añadir mas reglas, deben ser todas true) y or (permite añadir otra regla y solo una de las dos debe ser true para aplicar).

*Las medidas habituales para cambiar la disposición suelen medirse segun el "width" cuando es menor de 900px (tablets), cuando es menor de 600px (móviles) y cuando es mayor de 1200px (escritorio), más detalles aquí. Se pueden añadir mas de una regla utilizando el "and" entremedio, ejemplo: @media (max-width: 900px) and (min-width: 600px)
También se pueden crear diferentes archivos CSS para utilizarlos en diferentes anchos:
<link rel="stylesheet" media="screen and (min-width: 900px)" href="widescreen.css">
<link rel="stylesheet" media="screen and (max-width: 600px)" href="smallscreen.css">

Tips

Height de capa
Si una capa tiene contenido es mejor no definir su height sino que dejar que se adapte sola a su contenido.
Los div por defecto tienen display: block;
Los div tienen display:block por defecto, por lo que si queremos formar una sola columna no hace falta modificar el display. Por otro lado ocupan todo el ancho de la pagina, no hace falta especificar el width si no deseamos lo contrario.
Como centrar horizontalmente?
  • Si es un elemento "inline" (text, links, etc.) podemos utilizar text-align: center;.
  • Si es un un elemento "block" (div, p, etc.) podemos centrarlo con margin: 0 auto; siempre que hagamos definido un width, sino ocupará todo el ancho y ya se interpreta como centrado.
Como centrar verticalmente?
  • Si es un elemento "inline" (text, links, etc.) podemos utilizar el padding o si es un texto de una sola linia podemos utilizar el line-height: ... con el mismo valor que height.
  • Si se trata de multiples lineas de texto podemos utilizar vertical-align: middle; o definir el elemento contenedor como display:flex; y centrarlo con justify-content: center; flex-direction: column;, esto solo funcionará si el contenedor tiene definido un height.
Centrar vertical y horitzonal rapido
poner al padre height: 100%; display: grid; y al hijo margin: auto;
Hijos de Flexbox con margin-top: auto;
Si definimos margin-top: auto; en un elemento hijo de flexbox (column) este elemento se quedará en la parte inferior siempre, independientemente del contenido que tenga encima suyo o el espacio vacio. También podemos añadir margin: 0 auto; antes para centrarlo horizontalmente.
El footer abajo sin importar el height
Podemos utilizar Flexbox en el padre que contiene el footer y el resto de elementos de la página para enviar el footer siempre al final de la página tanto si se ocupa todo el height, haya scroll o no.
min-height: 100vh;
display: flex;
flex-direction: column;
justify-content: space-between;
Tener en cuenta width del Scrollbar
Importante saber que la medida "vw" incluye el espacio del scrollbar, por lo que si utilizamos 100vw el contenido quedara debajo del scrolbar. Para que el html no quede por debajo del scrollbar y parece que los elementos no queden centrados del todo incluso cuando cambiamos de una página con scrollbar a otra sin se nota el cambio de width. Hay varias maneras de solucionarlo:
  • html { overflow-y: overlay; }
  • html { overflow-y: scroll; }
  • html { width: 100vw; }
